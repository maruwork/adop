# ADOP Support

Status: Active

## Before Opening An Issue

1. Read `README.md`.
2. Run the smallest relevant command from `ADOP_GENERIC_QUICKSTART.md`.
3. Confirm whether the problem belongs in generic ADOP or a project-local overlay.
4. Check whether the expected behavior is already described in the checklist, template, or verification contract.

## Where To Ask

- reproducible defects: open a bug report
- lifecycle or schema proposals: open a feature request
- private repository preparation questions: include the exact step from `PRIVATE_HOST_BOOTSTRAP_RUNBOOK.md`

## What To Include

- command used
- relevant file path
- expected behavior
- actual behavior
- smallest useful log or artifact excerpt
