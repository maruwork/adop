ADOP governance dashboard

ADOP shows which external tools are under review, approved, blocked, retired, or kept only as history.

Read this page to understand the current decision. Open a row when you need the reason, the usage limits, or the exact next step. Most people only read this page; only the decision owner usually needs to run commands.

For operators

Run commands only when you are changing a decision

Use case key (scene) The label ADOP uses to keep one review context separate from another, such as lint-ci. Most readers only need to recognize it, not type it.
Allowed use area (landing target) Where an approved tool may actually be used in the workflow or system.
Usage limits (control model) The limits on use, especially around writeback, authority, and review.
