# Contributing to ADOP (Draft)

Status: Draft

This file is a pre-publication draft. It exists so the contribution surface can be made explicit before publication.

## Current Intent

- keep `ADOP` generic
- keep project-local current state and landing-target authority out of common authority
- treat artifact schema and lifecycle as shared core
- prefer bounded additions over broad workflow sprawl

## Expected Contribution Areas

- artifact schema and validation
- CLI usability and bounded verification
- generic checklist/template improvements
- public docs and export guidance

## Must Not Be Treated As Core Contributions

- project-local current trial boards
- project-local operator flows
- project-local landing target authority
- project-local adoption decision logs

## Before Opening Public Contributions

- license has a chosen default and must be reflected in the public repo
- host/owner has a chosen default and must be reflected in the public repo
- issue/PR policy has a chosen default and must be reflected in the public repo
- public CI expectations have a chosen default and must be reflected in the public repo
